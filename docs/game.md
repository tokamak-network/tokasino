### 1.3 공통 컴포넌트

#### 버튼

| 상태 | 스타일 |
|---|---|
| **Primary** | 배경 `--neon-cyan`, 텍스트 `--bg-primary`, 2px 픽셀 보더, 글로우 효과 |
| **Primary:hover** | 글로우 강도 1.5배, 0.9 → 1.0 scale 트랜지션 (150ms) |
| **Primary:active** | scale 0.95, 글로우 제거, 배경 밝기 80% |
| **Primary:disabled** | 배경 `--text-muted`, 글로우 없음, cursor: not-allowed |
| **Secondary** | 배경 투명, 보더 1px `--neon-cyan`, 텍스트 `--neon-cyan` |
| **Danger** | 배경 `--neon-pink`, 텍스트 white |

#### 카드/패널
- 배경: `--bg-secondary`
- 보더: 1px solid rgba(0,240,255,0.15)
- border-radius: 2px (픽셀 느낌 유지)
- 패딩: 24px
- hover 시 보더 rgba(0,240,255,0.3)으로 밝아짐

#### 인풋 필드
- 배경: `--bg-surface`
- 보더: 1px solid `--text-muted`
- focus 시 보더 `--neon-cyan` + 얇은 글로우
- 폰트: `VT323` (숫자 입력), `Galmuri11` (텍스트)
- 높이: 48px

### 1.4 배경 효과

| 효과 | 구현 | 성능 참고 |
|---|---|---|
| CRT 스캔라인 | `repeating-linear-gradient`, 2px 간격 반투명 검정 오버레이 | CSS only, 저비용 |
| 격자 패턴 | 40px 간격 1px `rgba(0,240,255,0.03)` 라인 | CSS background-image |
| 별/파티클 | Canvas 또는 CSS animation, 저밀도 (20~30개) | requestAnimationFrame |
| 스캔라인 sweep | 수직 라인이 3초 간격으로 위→아래 이동 | CSS animation |

> ⚠️ 모든 배경 효과는 `prefers-reduced-motion: reduce` 미디어 쿼리에서 비활성화

## 3. 화면별 상세 스펙

### 3.1 메인 게임 화면

이 화면이 서비스의 핵심이며, 사용자가 가장 많은 시간을 보내는 곳입니다.

#### 레이아웃 (위→아래)

```
┌──────────────────────────────────────┐
│         ① 상대방 선택 영역             │
│         (결과 전: "?", 후: 아이콘)      │
│              120×120px               │
├──────────────────────────────────────┤
│          ② VS 표시 영역               │
│     네온 글로우 "VS" (결과 전)          │
│     WIN / LOSE / DRAW (결과 후)       │
├──────────────────────────────────────┤
│      ③ 내 선택 영역 (가위/바위/보)      │
│       3개 버튼, 가로 배치              │
│     각 160×160px, 간격 32px          │
├──────────────────────────────────────┤
│        ④ 베팅 컨트롤 영역              │
│  [−] [금액 입력] [+]  [×2] [MAX]     │
│         [PLAY 버튼]                  │
├──────────────────────────────────────┤
│          ⑤ 룰렛 영역                  │
│   (승리 시에만 표시, 평소 숨김)          │
│     가로 스트립 룰렛, 480×80px         │
└──────────────────────────────────────┘
```

#### ① 상대방 선택 영역

- **대기 상태:** 120×120px 컨테이너 안에 `"?"` 텍스트 (픽셀 폰트 48px, `--neon-purple`, 부드러운 펄스 애니메이션)
- **결과 공개:** `"?"` → 상대방의 선택 아이콘으로 전환 (0.5초, 플립 애니메이션 Y축 회전)
- **아이콘 스타일:** 픽셀아트 스프라이트 (가위✌️/바위✊/보🖐), 각 120×120px, 4색 이하 도트 그래픽

#### ② VS / 결과 표시 영역

| 상태 | 표시 | 스타일 |
|---|---|---|
| 대기 | "VS" | `Press Start 2P` 36px, `--neon-cyan` 글로우, 느린 펄스 |
| 판정 중 | "VS" 깜빡임 | 100ms 간격 on/off 반복 (0.8초간) |
| 승리 | "WIN!" | `--neon-yellow` 강한 글로우, scale 1.0→1.3→1.0 바운스 (0.5초) |
| 패배 | "LOSE" | `--neon-pink`, 글로우 약하게, 살짝 흔들림 (shake 0.3초) |
| 무승부 | "DRAW" | `--text-secondary`, 글로우 없음, 페이드인 |

#### ③ 내 선택 영역 (가위 / 바위 / 보 버튼)

**각 버튼 스펙:**

- **크기:** 160×160px
- **배경:** `--bg-secondary`, 2px 솔리드 보더 `rgba(0,240,255,0.2)`
- **아이콘:** 중앙 정렬, 80×80px 픽셀아트 스프라이트
- **라벨:** 아이콘 하단 8px 여백, `Galmuri11` 14px, `--text-secondary`

| 버튼 상태 | 동작 |
|---|---|
| **기본** | 배경 `--bg-secondary`, 보더 약한 글로우 |
| **hover** | 보더 `--neon-cyan` 글로우 강해짐, scale 1.05 (150ms ease-out), 아이콘 밝기 120% |
| **선택됨** | 보더 `--neon-cyan` 2px + 강한 글로우, 배경 `rgba(0,240,255,0.08)`, 체크 뱃지 우상단 |
| **비선택 (다른 거 선택 후)** | opacity 0.4, 글로우 없음 |
| **비활성 (게임 진행 중)** | pointer-events: none, opacity 0.3 |

**선택 시 피드백:**
1. 클릭 즉시: 짧은 "삐" 효과음 (8bit 톤, 50ms)
2. 선택 보더 애니메이션 150ms
3. 다른 버튼 dim 처리 200ms

#### ④ 베팅 컨트롤 영역

**베팅 금액 입력 그룹:**

```
[−]  [ 0.10 ▾ ]  [+]    [×2]  [MAX]
              [▶ PLAY]
```

| 요소 | 크기 | 스타일 | 동작 |
|---|---|---|---|
| `−` 버튼 | 40×48px | Secondary, `VT323` 24px | 최소 단위만큼 감소, 길게 누르면 연속 감소 (200ms 간격) |
| 금액 입력 | 160×48px | 인풋 필드, `VT323` 24px, 우측 정렬 | 직접 입력 가능, 포커스 시 전체 선택, 숫자+소수점만 허용 |
| `+` 버튼 | 40×48px | Secondary | 최소 단위만큼 증가, 길게 누르면 연속 증가 |
| `×2` 버튼 | 56×48px | Secondary | 현재 금액 2배, 최대 한도 초과 시 최대값으로 |
| `MAX` 버튼 | 56×48px | Secondary | 허용 최대 베팅액 설정 |
| `PLAY` 버튼 | 280×56px | Primary, `Press Start 2P` 18px | 아래 상태 머신 참조 |

**PLAY 버튼 상태:**

| 상태 | 텍스트 | 스타일 | 조건 |
|---|---|---|---|
| 지갑 미연결 | "CONNECT WALLET" | `--neon-cyan` 배경 | 지갑 연결 안 됨 |
| 선택 필요 | "SELECT HAND" | disabled 스타일 | 가위/바위/보 미선택 |
| 금액 부족 | "INSUFFICIENT BALANCE" | disabled, `--neon-pink` 텍스트 | 잔액 < 베팅액 |
| 준비 완료 | "▶ PLAY" | Primary, 미세한 펄스 글로우 | 모든 조건 충족 |
| 진행 중 | "PLAYING..." | 스피너 아이콘 회전, disabled | 트랜잭션 처리 중 |

#### ⑤ 룰렛 영역 (승리 시에만 표시)

**구조:** 가로 스트립 룰렛. 배당 배수 칸들이 좌→우로 빠르게 스크롤 후 감속하여 정지.

- **컨테이너:** 480×80px, 중앙 정렬, `overflow: hidden`
- **뷰포트:** 중앙에 하이라이트 프레임 (80×80px, `--neon-yellow` 2px 보더 + 글로우)
- **칸(셀):** 각 80×80px, 배당 배수 텍스트 중앙 정렬

| 배당 | 셀 배경 | 텍스트 색상 | 출현 빈도 |
|---|---|---|---|
| ×1 | `#1A1A3E` | `--text-secondary` | 40% (가장 많음) |
| ×2 | `#1A2A3E` | `--neon-cyan` | 30% |
| ×4 | `#2A1A3E` | `--neon-purple` | 18% |
| ×7 | `#3A1A2E` | `--neon-pink` | 9% |
| ×20 | `#3A2A0A` | `--neon-yellow` | 3% |

**룰렛 애니메이션 시퀀스:**

| 단계 | 시간 | 동작 |
|---|---|---|
| 1. 등장 | 0 ~ 0.3초 | 컨테이너가 아래에서 슬라이드업 + 페이드인 |
| 2. 고속 회전 | 0.3 ~ 2.0초 | 셀이 초당 약 20칸 속도로 스크롤 (blur 효과 선택적) |
| 3. 감속 | 2.0 ~ 3.5초 | cubic-bezier(0.25, 0.1, 0.25, 1)로 점진적 감속 |
| 4. 정지 | 3.5초 | 결과 셀이 하이라이트 프레임 정중앙에 정지 |
| 5. 결과 강조 | 3.5 ~ 4.5초 | 정지 후 하이라이트 프레임 + 셀 함께 펄스 (scale 1.0→1.1→1.0, 2회) |

**배당별 결과 연출:**

| 배당 | 추가 연출 |
|---|---|
| ×1 | 기본 하이라이트만 |
| ×2 | 하이라이트 + 짧은 코인 이펙트 (3~5개 파티클) |
| ×4 | 하이라이트 + 코인 샤워 + 화면 살짝 흔들림 |
| ×7 | 화면 전체 플래시 + 코인 폭발 + "GREAT!" 텍스트 팝업 |
| ×20 (잭팟) | 아래 잭팟 연출 참조 |

**×20 잭팟 연출 시퀀스:**

1. **화면 플래시** (0~0.2초): 전체 화면 흰색 → 원래로 (opacity 0.8→0)
2. **"JACKPOT!" 텍스트** (0.2~1.0초): 화면 중앙, `Press Start 2P` 56px, `--neon-yellow` 강한 글로우, scale 0→1.3→1.0 바운스
3. **코인 폭포** (0.5~3.0초): Canvas 파티클, 화면 상단에서 수십 개 코인 쏟아짐
4. **배경 색상 변화** (1.0~3.0초): 배경이 일시적으로 `--neon-yellow` 색조로 변한 뒤 복귀
5. **금액 카운트업** (2.0~3.5초): 획득 금액이 0에서 실제 금액까지 빠르게 올라감 (`VT323` 36px)
6. **사운드:** 슬롯머신 잭팟 효과음 (8bit 스타일, 약 2초)

### 3.2 게임 진행 상태 머신

아래는 메인 게임 화면의 전체 상태 전이를 정의합니다.

```
[IDLE] → 가위/바위/보 선택 → [HAND_SELECTED]
[HAND_SELECTED] → PLAY 클릭 → [BETTING]
[BETTING] → 트랜잭션 승인 → [RESOLVING]
[RESOLVING] → 결과 수신 →┬→ 승리 → [RESULT_WIN]
                         ├→ 패배 → [RESULT_LOSE]
                         └→ 무승부 → [RESULT_DRAW]
[RESULT_WIN] → 룰렛 시작 → [ROULETTE_SPINNING]
[ROULETTE_SPINNING] → 룰렛 정지 → [ROULETTE_RESULT]
[ROULETTE_RESULT] → 3초 후 또는 클릭 → [IDLE]
[RESULT_LOSE] → 2초 후 또는 클릭 → [IDLE]
[RESULT_DRAW] → 1.5초 후 또는 클릭 → [IDLE]
```

**각 상태별 UI 변화:**

| 상태 | 선택 버튼 | PLAY 버튼 | 상대방 영역 | VS/결과 영역 | 룰렛 | 베팅 입력 |
|---|---|---|---|---|---|---|
| IDLE | 활성 | 조건별 | "?" 펄스 | "VS" | 숨김 | 활성 |
| HAND_SELECTED | 선택된 것만 활성 | "▶ PLAY" 활성 | "?" 펄스 | "VS" | 숨김 | 활성 |
| BETTING | 비활성 | 스피너 | "?" 펄스 빨라짐 | "VS" | 숨김 | 비활성 |

## 6. 반응형 및 접근성

### 6.1 브레이크포인트

이 프로젝트는 데스크탑 웹 우선이지만, 최소한의 반응형 대응을 포함합니다.

| 브레이크포인트 | 범위 | 대응 |
|---|---|---|
| Desktop (기본) | ≥1280px | 풀 레이아웃 |
| Desktop 소형 | 1024–1279px | 좌우 패딩 축소, 카드 간격 줄임 |
| Tablet | 768–1023px | LP 대시보드 2컬럼 → 1컬럼 스택 |
| Mobile | <768px | "데스크탑에서 접속해주세요" 안내 메시지 + 기본 플레이 기능만 제공 |

### 6.2 접근성 최소 요구사항

- 모든 인터랙티브 요소에 `aria-label` 제공
- 키보드 탐색: Tab 순서 (선택 버튼 → 베팅 입력 → PLAY), Enter/Space로 선택
- 포커스 링: 2px `--neon-cyan` outline, offset 2px
- 색상 대비: 텍스트 vs 배경 최소 4.5:1 (WCAG AA)
- `prefers-reduced-motion`: 모든 애니메이션 즉시 완료 상태로 전환, 파티클 비활성화
- 사운드: 기본 Off 상태, 유저가 명시적으로 활성화
- 스크린 리더: 게임 결과를 `aria-live="assertive"`로 알림 ("승리! 4배 배당")
